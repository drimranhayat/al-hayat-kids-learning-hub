# Al-Hayat Kids Learning Hub Website

This is the Phase 1 static website for Al-Hayat Kids Learning Hub. It is built for GitHub Pages, so it can be published without a server, database, or paid hosting.

## What is included

- Premium homepage with parent-focused conversion flow
- Program sections for Quran, Arabic, Islamic Studies, and Early Years
- Separate SEO pages for Quran Learning, Arabic for Kids, Islamic Studies, Online Classes, and Admissions
- Deeper parent-facing content on each SEO page, including concerns, learning stages, class rhythm, progress notes, and expanded FAQs
- Trust and child safety messaging without fabricated claims
- Free trial inquiry form that prepares a WhatsApp message
- SEO foundation for Islamic learning and Quran classes for kids
- Responsive mobile-friendly layout

## Files

- `index.html` - website content and page sections
- `styles.css` - responsive visual design system
- `script.js` - mobile menu and WhatsApp inquiry form
- `assets/hero-learning-hub.png` - generated hero image
- `ROADMAP.md` - next build phases and GitHub launch steps
- `quran-learning.html` - SEO page for Quran classes for kids
- `arabic-for-kids.html` - SEO page for Arabic classes for kids
- `islamic-studies.html` - SEO page for Islamic studies for kids
- `online-classes.html` - SEO page for online Quran and Islamic classes
- `admissions.html` - admissions and free trial page
- `sitemap.xml` - sitemap for search engines
- `robots.txt` - crawler instructions with sitemap location

## Before publishing

Update these placeholders:

1. In `script.js`, replace `923001234567` with the real WhatsApp number in international format.
2. In `index.html`, replace `hello@example.com` with the real email address.
3. Confirm the JSON-LD website URL matches the final website URL.
4. Add verified teacher names, credentials, policies, and real testimonials only after confirmation.

## Publish on GitHub Pages

1. Create a new GitHub repository named `al-hayat-kids-learning-hub`.
2. Upload all project files and folders to the repository.
3. Open repository settings.
4. Go to Pages.
5. Set the source to the main branch and root folder.
6. Save.
7. GitHub will publish the website at a `github.io` address.

## Recommended next step

After publishing, create separate pages for Programs, About, Admissions, FAQs, Blog, and Contact. This will improve SEO and make the platform easier to scale.
